# UI v49｜桌機可測試性與擴充檢驗包

本輪目標從「繼續收視覺」改成「盡快補到可測試／可檢驗」。

## 這輪新增

### 1. 桌機測試與檢驗面板
右側新增一塊桌機專用檢驗面板，直接整理出四條最常需要檢查的項目：
- 事件卡是否已揭示
- 任務卡是否已載入
- 行動卡是否可打開完整操作
- 回合紀錄是否已生成

每條都會直接顯示目前狀態，不需要自己在不同區塊之間來回比對。

### 2. 桌機快速檢驗按鈕
在同一塊面板內補了四個實際可點的入口：
- 打開事件全文
- 固定任務焦點
- 打開手牌完整操作
- 前往完整紀錄頁

這樣使用者可以更快做 smoke check，而不是只能被動等流程剛好走到某個 overlay。

### 3. 未來加卡的承接檢驗區
補了一塊「未來加卡的承接位」，直接把目前桌機 UI 所吃的三個 profile 顯示出來：
- 事件池 profile
- 任務池 profile
- 行動牌池 profile

目的不是 debug，而是提醒後續在新增事件卡／任務卡／行動卡時，桌機全文視窗與 metadata 區塊應繼續承接這些 profile 與 id，不要把 UI 寫死成只對當前種子資料有效。

下一輪若繼續往「可測試」推，最值得做的是：
1. 讓任務焦點從固定浮窗升級成真正的桌機全文 overlay
2. 補事件／任務／手牌的 footer 操作語言一致化
3. 補一條桌機 smoke test 專用流程帶（從事件全文 → 出牌 → 任務 → 紀錄頁）
