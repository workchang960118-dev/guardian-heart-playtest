# UI v44｜driver rail、事件／任務／能力條列提示、手機快捷帶擴充

## 本輪定位
延續 v43 的 control cockpit 主線，這輪不新增規則，而是把主畫面再往「單頁操作台」推深：
- 單頁指令台新增主流程導引卡（driver rail）
- 事件／任務／角色能力焦點卡補成條列式提示
- 手機底部快捷帶補更多高頻操作與下一步提示

---

## 一、單頁指令台新增主流程導引卡
在主流程快捷操作上方新增 `flowDriverLinesZh`：
- 若有 pending loss，明示先處理哪一席的損失反應
- 若目前在 campfire，指出目前停在哪個營火步驟
- 若事件尚未補滿，直接提示還缺多少 SR / SP
- 若任務已可宣告或仍被卡住，直接在這裡顯示

呈現方式改為 2 欄 driver cards，而不是只剩一條 tag + 一串小膠囊。

目的：
- 讓主畫面更像「現在先做什麼 → 下一步看什麼」
- 降低玩家還得自己去右側拼資訊的負擔

---

## 二、事件焦點卡補成可行動的條列提示
新增 `eventActionSuggestionsZh`：
- 先補需求 / 數值已補滿
- 多人投入門檻是否啟用、是否達成
- 未解懲罰直接列出

這些提示不是取代原本的 requirement / remaining pills，
而是補上一層「現在該怎麼理解這張事件」的中文說明。

---

## 三、任務焦點卡補宣告條件導引
新增 `taskFocusChecklistZh`：
- 任務摘要
- 第一條阻塞原因
- 現在是否在營火階段、能不能宣告

目的：
- 讓玩家不用只看按鈕 disabled 與任務 rail 猜規則
- 直接知道是「條件差一點」還是「時機還沒到」

---

## 四、角色能力焦點卡補機會窗口提示
新增 `abilityFocusChecklistUi`：
- 角色能力目前窗口
- 第一條 checklist 提醒
- 本輪可用次數 / 已用完 / 被 toggle 關閉

目的：
- 讓角色能力不只是名字＋可用/不可用 badge
- 更接近「這輪什麼時候該想到它」

---

## 五、手機底部快捷帶擴充
手機底部固定快捷列由 4 顆按鈕擴成 6 顆：
- 回到本人地格
- 查看事件
- 直接使用手牌
- 繼續營火
- 查看任務
- 補齊指定

底部說明區也改成雙卡：
- 一張顯示 blocker / 快捷提示
- 一張顯示現在主流程導引的第一句

目的：
- 讓手機版不用頻繁切回上方才能做常用操作
- 把主流程導引也前置到手機視圖

---

## 六、規則與 UI 對應
本輪沒有改動規則裁定；仍遵守：
- 任務只能在營火階段主動宣告
- 壓力 3 之後要檢查多人投入
- pending loss / discard / blocking window 仍優先於一般操作
- 角色能力仍只是顯示與提醒，不代替正式 validator 判定
