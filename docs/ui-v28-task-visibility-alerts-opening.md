# UI v28｜任務浮窗可見性、開場導入、危急警示與節奏提示

## 這刀主修
1. 任務列改成更穩定的懸浮閱讀窗
   - hover / focus / click 都能叫出較大的任務浮窗
   - 浮窗顯示條件、獎勵、目前狀態、阻塞原因
   - 營火且可宣告時，浮窗內直接出現「宣告完成」
   - 不再靠容易看不見的小 tooltip

2. 開場導入補回
   - 離開大廳後首次進局，先顯示「世界觀簡述 + 勝利條件」
   - 關閉後才回到局面並接事件揭示

3. 中央廣播與危急警示
   - 一般廣播移到畫面正中央，改深色字、無背景
   - 任一玩家 SR 或 SP = 1 時，畫面中央出現紅色持續閃爍警示，且不自動消失

4. 黑夜過場與即時反應辨識度
   - 黑夜過場改用漸暗漸亮 keyframes
   - 一般損失反應條加上脈衝閃爍
   - 關鍵損失的主按鈕也加強提示

5. 終局結算畫面
   - 改為內容區可滾動，避免內容過長時看不到下半部

6. AP 歸零提示
   - 主操作區的「結束回合」按鈕在 AP = 0 時會脈衝提示

## 備註
- 這包在容器內已排除 JSX 結構錯誤。
- 但由於這份 handoff 包沒有可用的 node_modules，且容器內也無法順利補齊 Next / React 依賴，因此這次沒辦法完整重跑 `next build`。
- 建議你安裝後先本地跑一次安裝與 build。
