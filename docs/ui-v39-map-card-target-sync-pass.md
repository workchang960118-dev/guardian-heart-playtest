# UI v39｜地圖選取 × 手牌指定地格同步收斂

## 這輪主題
把「指定地格」從原本仍偏表單式的操作，再往單頁主畫面推進一層：

- 點主地圖即可同步成手牌目標地格
- 地圖本身會顯示目前處於「出牌指定地格」模式
- 目標地格在地圖上有獨立高亮，不再只存在於側欄或 modal 文字
- 指定地格模式下，雙擊不再誤觸快速移動，而是只鎖定手牌目標

## 實作重點

### 1. 地圖舞台進入「指定地格模式」
`GuardianHeartMapStage` 新增：
- `cardTargetModeEnabled`
- `cardTargetCardNameZh`
- `cardTargetTileId`
- `onChooseCardTargetTile`

效果：
- 地圖頂部顯示目前正在替哪張牌指定地格
- 已選中的手牌目標地格會以紫色圈層高亮
- hover / focus 文案會改成出牌模式的描述

### 2. 地圖點選直接同步到手牌
在指定地格模式下：
- 單擊地格：同步為手牌目標
- 雙擊地格：只鎖定手牌目標，不快速移動
- 小浮層新增「指定為手牌地格」按鈕

這讓玩家不必先記地格名稱，再回手牌區下拉選單。

### 3. 快捷區與完整 modal 都明示同步狀態
手牌快捷區與完整手牌 modal 都補上：
- 目前鎖定的目標地格
- 是否已與主地圖同步
- 提示玩家可直接回主地圖點選

### 4. 目標 fallback 更穩
當前手牌若需要指定地格：
- 若原本指定值失效，會自動回退到可用地格
- 會優先沿用目前地圖焦點，其次才回退到排序後的第一個可用地格
- 切到需要指定地格的牌時，會優先帶入目前地圖焦點

## 使用者可感知的改變
- 指定地格不再像填表，而像直接在地圖上操作
- 單頁主畫面完成度更高
- 出牌時不容易把「選地格」和「移動」混在一起
- 桌機與手機版的三個主要地圖舞台都吃到同一套行為
