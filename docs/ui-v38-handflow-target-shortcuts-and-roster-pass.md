# UI v38｜手牌指定捷徑＋玩家列關係標示收斂

## 這輪重點
本輪延續 v37 的單頁操作方向，將「需要指定目標的手牌」往主畫面前推，降低每次都要進完整操作視窗的頻率，同時讓玩家列更直接顯示互助／位置關係。

## 本輪完成

### 1. 手牌完整操作視窗改成「可直接點選」優先
- 手牌操作視窗擴成 `max-w-3xl`
- 新增「送出前確認」區塊：
  - 顯示每個需求欄位是否完成
  - 顯示目前已選的隊友／地格／資源摘要
- 針對不同需求提供快捷指定：
  - 指定隊友：改成可直接點玩家 pill，並保留 select fallback
  - 指定地格：優先顯示目前位置／相鄰／關鍵站點，並保留完整 select
  - 指定資源：SR / SP 改為 segmented pill
  - 指定隊友投入資源：同樣改為直接點選
- footer 狀態文案改成會同時顯示：
  - 尚缺欄位
  - 或目前因規則不可送出的原因

### 2. 右側手牌區升級為「可先在主畫面補齊指定」
- 選中手牌後，不只顯示效果與 badge
- 若該牌需要指定：
  - 可直接在右側資訊欄快速指定隊友 / 資源 / 隊友投入資源 / 常用地格
- 若需求已補齊：
  - 可直接在主畫面按「直接使用」
  - 不再強迫先打開完整視窗
- 若仍想看完整說明：
  - 保留「細看操作」入口

### 3. 玩家列補上相對位置關係標示
- 玩家卡新增 `relationLabel`：
  - 本人
  - 同格
  - 相鄰
  - 其他
- 與原本的地格、陪伴狀態並排顯示
- 讓玩家更快判讀：
  - 哪些人可作為互助或指定隊友候選
  - 哪些人位置上更適合行動牌／支援牌

### 4. 選牌 fallback 行為更穩
- 開啟需要指定隊友的手牌時，若目前 target seat 不合法，會自動回退到第一個可選隊友
- 避免出現畫面看起來像有選人，但其實 state 不合法的狀況
