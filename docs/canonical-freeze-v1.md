## 二、目前已完成的第一階段排查結論
### 1. AI 不是小因子，而是大因子
- solver 型 AI 會把勝率推太高
- `canonical_humanized_v2f` 會把勝率壓太低
- 因此不能把所有問題都歸因於規則本身，也不能只說「AI 太聰明」

### 2. 事件層真正值得保留的修正是 A2
- **保留：事件未解懲罰 SP +1**
- **不列為目前主修：事件立即效果 SP +1**
- 事件需求 SP 側曾被驗出可疑，但目前最穩定、最適合作為主線單刀的是 A2

### 3. v6 過硬的主因更像 AI 基底太緊
- 在 `canonical_humanized_v2f` 基底上補 survival cadence 第二刀，整體過硬
- 在 `canonical_humanized` 基底下重新驗 A2，結果較接近首玩體驗帶

### 4. 現在不要同時亂動其他層
目前**先凍住、不作主修**的層：
- 任務要求
- 任務獎勵
- 地格核心效果
- 角色能力
- 壓力軸效果
- 手牌數量
- AP 上限

## 四、目前主線的判讀口徑
後續比較不再只看「誰最接近 Python benchmark」，而是同時看：

### 第一層：機制校正
- 哪一個因子一動就明顯改變結果
- 事件 SP 結構是否偏寬
- survival cadence 是否仍需補第二刀

### 第二層：首玩體驗校正
- 首玩勝率是否接近 **40%–45%**
- **R1–R3 敗局占比 ≤ 15%**
- 敗局是否主要集中在 **R5–R7**
- **R6–R7 惜敗占比**是否夠高
- 勝局中是否常出現至少一次 **SR 或 SP ≤ 1** 的緊張時刻

---

## 五、目前已知需要保留的備註
- A2 的作用比較像：**提高未解風險，迫使 AI 更願意優先補事件，進而把其他地方榨緊**
- 目前敗局主因仍以 **SR / SP 歸零** 為主，不是壓力爆表
- 任務池中 **「支援網成形」** 目前屬低命中觀察任務，之後若做任務池調整，優先回頭檢查這張

## 六、之後若要新增事件卡／任務卡，怎麼處理
不是推翻目前結論，而是：

1. 將新牌池升版，例如：
   - `canonical_event_pool_v2`
   - `canonical_task_pool_v2`
2. 重跑關鍵 compare：
   - baseline
   - 主線 compare
   - 關鍵 regression check
3. 不必把整個研究史從 v1 全部重跑一遍

白話版：
- **結果要重跑**
- **方向性結論不用全部推翻**
