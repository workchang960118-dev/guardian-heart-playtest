# UI v15｜事件揭示、目標入口、右側爆閃修正、模擬頁摘要優先

這輪調整重點：

1. 房間主畫面補上全體事件揭示彈窗
   - 每輪翻出新事件後，所有玩家都會看到同一個事件揭示彈窗
   - 顯示事件名稱、事件敘述、解決條件、未解懲罰

2. 房間主畫面補上「本局目標」與「世界觀」入口
   - 頂部流程列新增兩個按鈕
   - 本局目標：說明勝利／失敗條件與目前進度
   - 世界觀：用短文說明 SR / SP / AP 與這局的情境定位

3. 修右側地格 hover 爆閃
   - 地圖操作選單不再由 hover 觸發，改成 click 選取後開啟
   - 右側地格的操作選單強制往左側開，減少貼齊邊界時的抖動與反覆開關
   - 預設不再自動選取任一地格，減少一進房就跳地圖操作框

4. 模擬頁改成摘要優先
   - 在報表頁最上方新增「先看這裡」摘要區
   - 直接告訴使用者：先看正式候選、首玩體驗基準、差距主因即可
   - 保留細節表格給進階分析時再看

5. 補強重複 key 風險
   - 模擬頁多個列表 key 改成資料 id + index

這版目標：
- 讓玩家主畫面更接近單頁、單視窗完成
- 讓模擬頁不用先懂整頁統計，也能先讀懂結論
- 讓其他 AI 在檢視這個 zip 時，更容易理解目前 UI 口徑與改動方向
