## 目前最值得持續盯的 3 個事件點

### 1. event_4_rescue_fatigue
這張目前仍不是完整真人版口徑。

原因：
- 規則文字是「每名玩家各自選擇失去 1SR 或 1SP」
- 目前實作是系統代選較低資源那一軸，並在摘要中明示

這對 simulation 是可接受的，但對真人局仍屬簡化版。

### 2. event_5_isolation_night
這張最依賴地圖與相鄰判定。

若後續：
- 地圖 adjacency
- token 位置
- 街巷信使免費移動
有任何 bug，這張的立即效果會最容易先失真。

### 3. event_2 / event_8 的 two_distinct_players
目前已從固定打前兩位，改成系統隨機抽目標。

這比舊版乾淨，但若之後要更接近真人局，仍可考慮：
- host 指定
- 或 AI 策略指定

## 這份表和 simulation debug 的對應方式

之後在 simulation compare / debug 中，如果看到：
- 某張事件解決率顯著偏低
- 某張事件抽到次數高但勝率影響特別大

就回頭對這張表，先看：
1. 它的立即效果是否已完整接入
2. 它是否仍有簡化口徑
3. 它是否特別依賴地圖 / 相鄰 / target selection

這樣才能分辨：
- 是平衡問題
- 還是規則口徑還沒對齊
